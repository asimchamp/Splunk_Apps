Modular Alert Framework | RedAlert
for Splunk

RedAlert extends the alerting capabilities of Splunk by providing a simplified and comprehensive configuration interface for alert actions.  In essence, RedAlert is like a Tag Management solution for alerts, giving your alert actions easy access to fields, search results, and alert arguments.  Searches come to life and become actionable as their alerts issue shell commands and REST searches in response to events in the infrastructure.  Alert Actions become auditable, with monitoring and logging of performance details, execution status, and errors.  Use RedAlert to automatically file incidents with your CMDB, restart broken applications, lockdown ACLs in response to threats, or page you, all at the click of a button...  or two...

Configuration is simple:

  1. Install RedAlert on a Search Head.

  2. Copy $SPLUNK_HOME/etc/apps/alerts/bin/redalert.sh to $SPLUNK_HOME/bin/scripts/redalert.sh -- use redalert.bat for Windows.

  3. [Optional]  Configure _passwd_.conf with Splunk credentials for REST API access.  This is only necessary to support alert actions that run searches; shell commands don’t need this.

  4. Define your alert actions in RedAlert's “Configuration” dashboard.

  5. Save or Edit an alert in Splunk.  To associate your alert with a RedAlert action, place the RedAlert action's ID in brackets, somewhere in the name of your Search/Alert.  For instance, a RedAlert action whose ID is “SEARCH_SAMPLE” would match any search/alert that has "[SEARCH_SAMPLE]” in its name.

  6. Select to "Run a Script" as an action for your alert.  Type "redalert.sh" as the filename to run.  (See Diagram 1) -- use redalert.bat for Windows.

  7. Go to the “Overview” dashboard and monitor your alerts.

* Use the `RedAlert` macro to search your audit logs for alert execution details.

NOTE:  RedAlert will configure a Data Input (UDP:9020) that listens only for "localhost" traffic.  This is used to capture and index performance and execution details for alerts.  If this data needs to be indexed at another location, be sure to configure the server to forward this data.

Troubleshooting: 

After configuring your alerts, check the Overview dashboard inside the app for status messages.  The "Interesting Events" will show you alert errors, as well as errors captured in _internal:web_services.log and _internal:splunkd.log.
