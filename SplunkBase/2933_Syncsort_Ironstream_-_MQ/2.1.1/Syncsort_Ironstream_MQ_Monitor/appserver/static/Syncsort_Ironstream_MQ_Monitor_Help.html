Syncsort Ironstream MQ Monitor

Mainframe Data Source: SMF 116 Records

The Syncsort Ironstream MQ Monitor application allows for the monitoring of WebSphere MQ queue information to assist with supporting critical business services. It provides insight into key performance metrics to determine if business services are being met or impacted. Multiple inputs allow users to identify issues at a high level, and drill down for more granular visibility into connection, base queue, and queue information.

WebSphere MQ accounting data is written as SMF 116 records. This application uses the SMF 116 Subtype 1 record, which provides accounting data for each task at thread and queue level.

The following SMF 116 Subtype 1 record fields are used within this dashboard:

Field Explanation
BASEAME Object base name
CLOSETIME Last MQ close time
GETCT Total CPU time for MQ gets
GETET Total elapsed time for MQ gets
GETN Number of MQ get calls
MAXQDPTH Maximum encountered queue depth
OBJNAME Object name
OPENTIME First MQ open time
PUTCT Total CPU time for MQ puts
PUTET Total elapsed time for MQ puts
PUTN Number of MQ put calls passed directly to waiting getter
WTIDCCN Connection name
