Data Model

Previous: Search Experience

Next: Pivot

Data models enable users to create Pivot reports and dashboards through the user interface without having to design complex searches. They are composed of objects that structure your data through simple searches, transactions, calculated fields, lookups, user-defined extractions, and more. With data models, you can generate a collection of searches that can be reused and distributed.

Building a Data Model

Data models are designed by Splunk knowledge managers: people who understand the format and semantics of their indexed data, and who are familiar with Splunk's search language. Data models are built to support the Pivot feature and enable users who are not familiar with the search language to get information out of Splunk

You create data models by:

  • Entering Pivot and (on the Select a Data Model page) clicking Manage Data Models. This takes you to the Data Models page, where you click New Data Model.
  • Navigating to System > Data Models to get to the Data Models page. Once there, click New Data Model

    • Once you create a data model you need to add Objects to that data model. There are five types of Objects you can create:

    Root event objects:

    They represents a set of data that is defined by a constraint: a simple search that filters out events that aren't relevant to the object

    Root search objects:

    Enable you to create object hierarchies where the base dataset is the result of an arbitrary transforming search--a search that uses statistical commands to define a base dataset where one or more fields aggregate over the entire dataset.

    Root transaction objects:

    These objects enable you to create object hierarchies that are based on a dataset made up of transaction events.

    Child objects:

    A child object inherits all of the constraints and attributes that belong to its parent object. When you define a new child object, you give it one or more additional constraints, to further focus the dataset that the object represents.
