ALERTING FILE-SYSTEM GLOBAL EXCLUSION

The alerting file-system global exclusion management interface allows you to configure Nmon alerting features for:

  • global exclusion of file-systems from alerting
    •

    file-systems exclusion:


  • mount: mount point of the Unix file-system
  • exclude: (true/false) explicit exclusion (default)
    •


    Global entries will be applied for all servers.

    You can use wildcards to match multiple file-systems at once, and case is insensitive.


    Lookup access:

    | inputlookup nmon_alerting_filesystem_global_exclusion

    Exclusion table - currently configured exclusions

    The exclusion table shows exclusions currently configured. To search for an entry, fill the filters and press enter. To pre-fill for addition, modification and deletion, click on any row of the table

    You can add a new exclusion entry, mount point is case insensitive and accepts wildcards to match multiple mount points at once

    CAUTION: any global exclusion will be applied to all servers!
