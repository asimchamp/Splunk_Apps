ALERTING THRESHOLD MANAGEMENT

The alerting threshold management interface allows you to configure Nmon alerting features for:

  • CPU usage monitoring for continuous configurable peaks in seconds
  • Physical memory usage monitoring for continuous configurable peaks in seconds
  • Virtual memory usage monitoring for continuous configurable peaks in seconds
    •

    By default, the alerting reports run for all servers with the following parameters:

  • 90 % of usage for CPU and Physical memory usage
  • 40 % of usage for Virtual memory usage
  • 5 minutes (300 seconds) of continuous peak
    •

    Alerting reports use KVstore alerting collections to allow a per server configuration:

    CPU usage:


  • alert_cpu_max_percent
  • alert_cpu_min_seconds
    •

    Physical memory usage:


  • alert_physical_memory_max_percent
  • alert_physical_memory_min_time_seconds
    •

    Virtual memory usage:


  • alert_virtual_memory_max_percent
  • alert_virtual_memory_min_time_seconds
    •


    When a server is matched within the nmon_alerting_threshold KVstore lookup, the server thresholds settings will override default alerting parameters.

    Per server alerting entries have the highest priority against any other settings.


    Lookup access:

    | inputlookup nmon_alerting_threshold
