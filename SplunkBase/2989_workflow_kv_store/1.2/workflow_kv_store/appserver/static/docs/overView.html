About Workflow kv Store App

This application will give you a general idea about the way to use kv store with Splunk.

In order to explain how to work the workflow kv store app, i invit you to visit this link and read the tuto


Tutorial Splunk for Kv store

In this tutorial, they recommand to convert the dashboards to HTML but it's really necessary you can use XML dashboards

So the workflow kv store contain only 3 dahsboards :

  • Starter
  • In progress
  • Validation

The first one is "Starter" dashboard, it used to get or add new elements within data source

I added an event click using JavaScript to select items and stored into another kv collection named editcollection. this collection is used by dashboard In progress.

the second one is "In progress" dashboard. He implement the content dashboard with "editcollection" lookup and now you can add some description or any other input type just don't forget that on transform.conf. So now you can add action on selected items (send to validation dashboard, remove or cancel all selection). When you send to validation dashbord you can see what are added on validate table.

Maybe somebody ask the question, what is the back table in this dashboard? "don't worry we will return after"

Finally the last dashboard is validation. Here you can confirm selected items or not. You can use permission on dashboard to allow for privileged persons like an Admin.

If the any item should go back to edition statement you can use the button "back to edition" and now you have something with back table on In progress dashboard.

You can perform your item's status with color code or label. In this App i added this on Starter dashboard.
