Here is how you index the data:

  1. Go to http://www.transtats.bts.gov/DL_SelectFields.asp?Table_ID=236&DB_Short_Name=On-Time
  2. Select which month of data you wish to index via the dropdown lists
  3. Check Prezipped File (Note that this App has been configured to only recognize data that includes all the fields made available)
  4. Click Download and Unzip Your File
  5. Now go to the Splunk indexer via Manager>Data Inputs>Add data>From files and directories>Skip Preview
  6. Select "Upload and index a file" and select the unzipped .csv file
  7. Important: Click on more settings and select manual from Source Type. Enter in "airline_csv" without the quotes
  8. Click save and after a few minutes, your data will be indexed correctly!
  9. Return to the FAA App and the dashboards and saved searches will now populate with data
  10. If you choose to, you can index more data from different months but note that your searches will become slow as this is a huge dataset. Use the TimeRange picker to adjust what time range of data you want to search on to improve performance
Alternatively, you can put the unzipped .csv files in $SPLUNK_HOME/etc/apps/FAA/flightdata/ and the application will automatically start indexing the data for you.
If you don't want to manually download the files, you can also run faascript.py located in the same folder. It will download and extract a range of files for you. Here is how to run it: python faascript.py 1/2011 2/2012 (Note: only tested on unix systems.) This will download all files from January 2011 to February 2012 into that folder and splunk will start indexing them. You can also just provide one date to download just one month of data. If this method does not work, just manually download the files and follow the steps above.
