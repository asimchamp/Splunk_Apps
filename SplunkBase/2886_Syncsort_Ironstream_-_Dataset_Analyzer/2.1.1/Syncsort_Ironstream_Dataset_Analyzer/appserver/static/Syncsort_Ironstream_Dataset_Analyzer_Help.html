Syncsort Ironstream® Dataset Analyzer

The Dataset Analyzer application allows for monitoring of job and user information related to specific datasets. Users of the app can define and track access to critical datasets within z/OS to determine potential security threats based on unauthorized access attempts.

Mainframe Data Source: SMF 14, SMF 15, SMF 17, and SMF 30 Records

SMF 14:

Record type 14 is written for non-VSAM direct access datasets, tape datasets, or virtual I/O datasets that are defined by DD statements or dynamic allocation and opened for RDBACK or INPUT processing by problem programs. The record is written when the dataset is closed or processed by the End of Volume (EOV).

SMF 15:

Record type 15 is written for non-VSAM direct access, or virtual I/O tape data sets that are defined by DD statements or dynamic allocation and opened for OUTPUT, UPDAT, INOUT, or OUTIN processing by problem programs. It is written when a dataset is closed or processed by the End of Volume (EOV)

SMF 17:

Record type 17 is written when a non-temporary DASD dataset or temporary DASD dataset is scratched. This record contains the dataset name, number of volumes, and volume serial numbers. The job name, time, and date that the reader recognized the JOB card for a particular job constitute the job log identification.

SMF 30:

Record type 30 provides job accounting information. It consolidates data that is found in SMF record types 4, 5, 20, 34, 35, and 40, and it provides additional information. SMF writes record type 30 when a work unit (such as a TSO/E session, started task, or batch job) starts or completes. A type 30 record is also written at the end of an SMF interval, if interval accounting is turned on.

In addition to the SMF records, Splunk users will need to create a .csv file entitled “CriticalDatasetCSV.csv.” This dataset is merely a one column table, each row with a dataset name, which represents a dataset to track. The single column should be titled “Dataset.”

This dashboard uses the above records to examine the following metrics:

  • Dataset Name
  • Dataset Access Request Time
  • Type of Dataset Access
  • Name of Accessing Job
  • Job Step Information
  • Job Completion Code
  • Program Name
