IP Reputation App

This app gives you the ability to correlate  realtime and historical machine data with threat data from honeypot project database to get a threat scoring of an IP. The script can be modified so that it requests from other databases (http://www.dnsbl.info/dnsbl-list.php)




Sample: Machine Data is collected from Webservers and processed in Splunk. Splunk will perform via external lookup the score of the IP-Addresses at Project Honey Pot Database. This database monitors over 110 Million IP’s, 112 Million Spam Traps and identified Millions of bad Networks and Systems. It has over 3000 harvesters who collect those data currently


Configuration

The threat score is pulled down from the project honey pot threat intelligence database. To access this database you’ll need to register for free and obtain an authorization code.

  1. Register and optain http:BL code:
    • https://www.projecthoneypot.org/services_overview.php
  2. Configure http:BL Key
    • go into $SPLUNK_HOME/etc/apps/ipreputation/bin/
    • open the scorelookup.py script with an editor
    • paste your Code into the “key” value and save the script
                 
  1. Some Dashboards are only supported in Splunk V6


Using threatscore lookup

Go into the Search app and test against your webserver logs. It’s recommended to group the ip’s in front of the lookup to boost performance and avoid multiple lookups of the same IP.

….| lookup threatscore clientip as your-field-to-lookup

Once executed you should have a additional field called threatscore. 0 means no entry in the database or the IP is okey. A higher number means a more dangerous IP.

You can also rename your lookup field to clientip if it has another name. | rename myiptag as clientip
