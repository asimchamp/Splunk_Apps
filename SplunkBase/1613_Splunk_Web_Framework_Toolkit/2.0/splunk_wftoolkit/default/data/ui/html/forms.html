Forms

Form views display common elements of web forms (such as a group of radio buttons or checkboxes, a dropdown list, or a text input). All of these views have a val method that can be used to retrieve their current value. For example: 

        splunkjs.mvc.Components.getInstance("example-textinput").val();

Events

All form views have a change event:

change: Fired when the value changes.

Universal options

All form views have a disabled option:

disabled: Disables the view object when set to true. Enables the view when set to false.

Here's an example:

RadioGroup view

The RadioGroup view represents a group of radio buttons. This view makes it easier to populate a radio button group from a search, but it can also be used like a standard HTML radio button group.

The radio group is set to: 

        var RadioGroupView = require("splunkjs/mvc/radiogroupview");
        new RadioGroupView({
                id: "example-radiogroup",
                default: "One",
                el: $("#divToHangOn")
        }).render();

Options

You can set properties for radio button groups in Javascript.

choices: A static dictionary of options for the radio button group (must be set in JavaScript). 

        // Set up choices            
        var choices = [
                {label: " One", value: "One"},
                {label:" Two", value: "Two"}, 
                {label:" Three", value: "Three"}];
        
        // Assign them to the button group
        splunkjs.mvc.Components.getInstance("example-radiogroup").settings.set("choices", choices);

managerid: The search manager to use for populating the radio button options. Set valueField, labelField, or both to specify which fields in the search results to use for values and labels (see the example below).

valueField: The search field to use for radio button values.

labelField: The search field to use for radio button labels. If a labelField is not set, the radio button labels default to the valueField values.

default: The default radio button option.

Events

After a change event, the RadioGroup view returns the value of selected radio button. 

        var rbg = splunkjs.mvc.Components.getInstance("example-radiogroup");
        var rbResult = document.getElementById("example-radiogroup-result");
        rbg.on("change", function(){
                rbResult.innerHTML = rbg.val();
        });

Example

In this example, a radio button group is created and bound to a search manager that shows all of the available indexes on the Splunk instance.

The radio group set is set to: 

        var RadioGroupView = require("splunkjs/mvc/radiogroupview");
        new RadioGroupView({
                id: "example-radiogroup",
                default: "One",
                managerid: "indexes-search",
                labelField: "index",
                valueField: "index",
                el: $("#divToHangOn")
        }).render();
        
        var SearchManager = require("splunkjs/mvc/searchmanager");
        new SearchManager({
                id: "indexes-search",
                search: "| eventcount summarize=false index=* index=_* | dedup index | fields index | head 5",
        });

Dropdown view

The Dropdown view represents a dropdown list that can be bound to a search manager, but can also be used as a standard HTML select element that emits change events.

Here's an example:

You have selected: 

        var DropdownView = require("splunkjs/mvc/dropdownview");
        new DropdownView({
                id: "example-dropdown",
                default: "One",
                el: $("#divToHangOn")
        }).render();

Options

You can set properties for a dropdown list, but some properties can be set in JavaScript.

choices: A static dictionary of options for the dropdown list (must be set in JavaScript): 

        // Set up choices            
        var choices = [{label: " One", value: "One"},
        {label:" Two", value: "Two"}, 
        {label:" Three", value: "Three"}];
        
        // Assign them to the dropdown
        splunkjs.mvc.Components.getInstance("example-dropdown").settings.set("choices", choices);

valueField: The search field to use for list options.

labelField: The search field to use for list options. If a labelFieldis not set, the dropdown options default to the valueField values.

default: The default dropdown option.

Events

After the change event, the Dropdown view returns the value of selected option. 

        var sbg = splunkjs.mvc.Components.getInstance("example-dropdown");

TextInput view

The TextInput view represents a basic text box. The val method returns the current text.

Options

The TextInput view only has one option:

default: The default text.

Example

In this example, two text boxes are created. The JavaScript shows how to create a change event handler that swaps the text box values when they change. 

        var TextInputView = require("splunkjs/mvc/textinputview");
        new TextInputView({
                id: "example-textinput1",
                default: "type here",
                el: $("#divToHangOn")
        }).render();
        
        new TextInputView({
                id: "example-textinput2",
                default: "to be replaced",
                el: $("#secondDivToHangOn")
        }).render();

Here's the change event handler in JavaScript: 

        // Get textinput elements
        var tb1 = splunkjs.mvc.Components.getInstance("example-textinput1");
        var tb2 = splunkjs.mvc.Components.getInstance("example-textinput2");
        
        // Attach change events
        tb1.on("change", function(){
            tb2.val(tb1.val());
        })
        tb2.on("change", function(){
            tb1.val(tb2.val());
        })

Password boxes

The TextInput view can be configured to show text as asterisks when you want a text box for entering passwords and other data that should not be displayed. Just set the TextInput's type property to "password".

Here's an example that creates a regular text box and a password text box. The JavaScript below that shows that when a change event occurs in one, the value is copied into the other. 

        var TextInputView = require("splunkjs/mvc/textinputview");
        
        new TextInputView({
                id: "example-passwordinput1",
                type: "password",
                default: "to be replaced",
                el: $("#divToHangOn")
        }).render();
        
        new TextInputView({
                id: "example-textinput3",
                el: $("#secondDivToHangOn")
        }).render();

Here's the change event handler in JavaScript: 

        // Get textinput elements
        var pb1 = splunkjs.mvc.Components.getInstance("example-passwordinput1");
        var tb3 = splunkjs.mvc.Components.getInstance("example-textinput3");
        
        // Attach change events
        pb1.on("change", function(){
            tb3.val(pb1.val());
        });
        tb3.on("change", function(){
            pb1.val(tb3.val());
        });

Checkbox view

The Checkbox view represents a Boolean value. The val method returns the current state of the checkbox (true or false).

Options

The Checkbox view only has one option:

default: The default value. Any value other than an empty string evaluates to true.

Example

This example shows how to create a Checkbox view. The JavaScript shows how to create a change event handler that displays the value of the checkbox.

Checkbox:

The checkbox is set to: false 

        var CheckBoxView = require("splunkjs/mvc/checkboxview");
        
        new CheckBoxView({
                id: "example-checkbox",
                el: $("#divToHangOn")
        }).render();

Here's the change event handler in JavaScript: 

        //get checkbox element and results line
        var cb1 = splunkjs.mvc.Components.getInstance("example-checkbox");
        var cb2 = document.getElementById("example-checkbox-result");
        
        //attach change event
        cb1.on("change", function(){
            cb2.innerHTML = (cb1.val() ? "true" : "false");
        });

CheckboxGroup view

The CheckboxGroup view represents a group of checkboxes. This view makes it easier to populate a checkbox group from a search, but it can also be used like a standard HTML checkbox group.

This example shows how to create a CheckboxGroup view. The JavaScript shows how to create a change event handler that displays the values of the selected checkboxes.

You selected: 

        var CheckboxGroupView = require("splunkjs/mvc/checkboxgroupview");
        new CheckboxGroupView({
            id: "example-checkboxgroup",
            default: "One",
            el: $("#divToHangOn")
        }).render();

Here's the change event handler in JavaScript: 

        var cbg1 = splunkjs.mvc.Components.getInstance("example-checkboxgroup");
        var cbgr = document.getElementById("example-checkboxgroup1-result");
        
        // Set choices (see next section for setting choices)
        
        // Attach change event.  Note: CheckboxGroup's result is always a javascript array.

                cbgr.innerHTML = cbg1.val().join(", ");
        });

managerid: The search manager that specifies the search to populate the checkbox group. Set valueField, labelField, or both to specify which fields in the search results to use for values and labels (see the example below).

valueField: The search field to use for checkbox values.

labelField: The search field to use for checkbox labels. If a labelField is not set, the checkbox labels default to the valueField values.

default: The default checkboxes.

Single view

The Single view displays a single value based on a search manager, which can be useful for showing an aggregate value such as a stats count. 

        var SingleView = require("splunkjs/mvc/singleview");
        new SingleView({
            managerid: "example-single-search",
            el: $("#divToHangOn")
        }).render();
        
        var SearchManager = require("splunkjs/mvc/searchmanager");
        new SearchManager({
            id: "example-single-search",
            search: "index=_internal | stats count",
        });

Options

You can set the following properties using JavaScript:

managerid: The search manager.

afterLabel: A label to be placed after the value.

beforeLabel: A label to be placed in front of the value.

Field: The search field to use as the display value.

classField: When set to range, works with the rangemap search command to visually convey severity (see the example below). For more on rangemap, see Using rangemap with single value panels.
